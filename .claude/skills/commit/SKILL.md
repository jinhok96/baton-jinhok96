## 커밋 메시지 형식

`{Prefix}: 설명`

| Prefix     | 용도                                                        |
| ---------- | ----------------------------------------------------------- |
| `Feature`  | 새 기능 추가, 기능 변경                                     |
| `Fix`      | 버그 수정                                                   |
| `Refactor` | 기능 변경 없는 코드 개선                                    |
| `Style`    | UI/CSS 스타일 변경                                          |
| `Content`  | 콘텐츠 추가/변경                                            |
| `Document` | 콘텐츠를 제외한 문서 추가/변경 (README, .claude/ 등)        |
| `Test`     | 테스트 코드 추가/변경                                       |
| `Chore`    | 배포 설정, 의존성 업데이트, 기타 설정 변경                  |

## 작업 완료 시 커밋 제안 형식

변경 파일을 유형에 따라 여러 커밋으로 분리. 각 커밋은 아래 3개의 독립 코드 블록으로 표시:

커밋 메시지:

```text
{Prefix}: 설명
```

git add 명령:

```text
git add file1 file2
```

파일 목록:

```text
- file1.html
- file2.css
```
